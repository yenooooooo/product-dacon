## 1. 프로젝트 개요

- **목표**: 제주도 버스 정류장의 18~20시 저녁 승차인원 예측
- **평가지표**: RMSE (Root Mean Squared Error)
- **학습 데이터**: 2019년 9월 (415,423행)
- **예측 대상**: 2019년 10월 1~16일 (228,170행)

### 핵심 난제
- 학습은 9월, 예측은 10월 → 월이 바뀌면서 공휴일, 날씨가 달라지는 시간적 갭
- 타겟의 71%가 0명 → 극단적으로 쏠린 분포
- 소수 정류장에서 최대 272명 → 이상치 존재

## 4. 특성 엔지니어링 — 팀원 5명 통합

### 피처 통합 결과
- 5명이 독립적으로 만든 피처의 **80%가 겹침** → 올바른 방향의 증거
- 기존 55개 (공통) + 팀원 고유 4개 = **최종 59개**

### 고유 피처 4개
| 피처 | 담당 | 설명 |
|------|------|------|
| `usertype_tourist` | 팀원 4 | BTS 카드 기반 관광객 비율 |
| `usertype_local` | 팀원 4 | BTS 카드 기반 현지인 비율 |
| `dist_to_dong` | 팀원 4 | 제주시 중심까지 거리 (km) |
| `passed_days` | 팀원 5 | 9/1 기준 경과일수 |

### 가장 중요했던 피처 3가지
1. **bts_morning_getoff**: 아침에 여기서 내린 사람 = 저녁에 여기서 다시 타는 사람
2. **bus_interval**: 버스가 얼마나 자주 오는가 (공급 빈도)
3. **net_flow**: 승차 - 하차 = 이 정류장이 출발지인가 도착지인가

## 5. 모델링 발전 과정

### 시행착오에서 얻은 교훈

| 단계 | 피처 | 모델 | 결과 | 교훈 |
|------|------|------|------|------|
| plan9 | 55개 | LGB only | 단독 최고 | 55개가 최적 피처 수 |
| plan10~16 | 57~95개 | LGB only | 전부 하락 | 피처 추가 = 과적합 |
| plan17 | 95개 | LGB + RF | 개선 | 모델 다양성이 핵심 |
| plan18 | 55개 | LGB + RF | 더 개선 | 55개 + 다양성 = 최적 |
| plan22 | 55개 | LGB + RF + XGB | 최고점 갱신 | 3모델 + 시간기반 튜닝 |

**핵심**: 피처를 늘리는 건 과적합을 유발하지만, 모델을 다양화하는 건 성능을 올린다.

### 손실함수 선택
- 대회 평가지표는 **RMSE**이지만, 학습 손실함수는 **MAE(L1)**를 사용
- 데이터의 71%가 0이고 극단값이 많아서, RMSE로 학습하면 큰 값에 끌려감
- 이상치에 덜 민감한 MAE + log1p 조합이 RMSE 점수에서도 더 좋은 결과

## 6. 최종 블렌딩 전략

### 3-Way 블렌딩 (Private 13위)

```
submission_team1_3way.csv =
    55개 피처 모델 × 0.5  (안정적 베이스)
  + 59개 피처 모델 × 0.2  (이용자 유형 보완)
  + 95개 피처 모델 × 0.3  (다른 관점의 예측)
```

### 왜 이 조합이 최고였는가
1. **55개 모델**: 과적합 없는 안정적 예측 → 비중 50%
2. **59개 모델**: 관광객/현지인 구분 정보 보완 → 비중 20% (55개와 상관 0.999라 소량)
3. **95개 모델**: LGB+RF로 다른 결정 경계 → 비중 30% (다양성 기여)

**최고 점수는 하나의 강력한 모델이 아니라, 서로 다른 관점을 가진 여러 모델의 조합에서 나왔습니다.**

## 7. 핵심 인사이트 5가지

1. **EDA가 피처의 근거** — 근거 없이 만든 피처는 노이즈가 됨
2. **피처 품질 > 피처 양** — 55개가 95개보다 단독 성능이 좋았음
3. **팀원 80% 겹침 = 올바른 방향** — 독립적 분석이 같은 결론 = 강력한 검증
4. **모델 다양성이 블렌딩의 핵심** — 같은 모델 여러 개보다 다른 모델 조합이 효과적
5. **3-Way 블렌딩** — 서로 다른 관점의 모델을 결합하면 단독보다 강력
