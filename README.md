## 나의 역할과 기여

### 모델링 전 과정을 설계하고 리드했습니다

팀원 5명이 각자 EDA를 진행하는 동안, 저는 **베이스라인 모델을 잡고 실험을 반복**하는 역할을 맡았습니다. plan1부터 plan24까지 총 24번의 실험을 거치면서 최적의 피처와 모델 조합을 찾았고, 이 과정에서 얻은 교훈이 최종 전략의 근간이 되었습니다.

### 팀원 5명의 피처를 분석하고 통합했습니다

각 팀원이 독립적으로 만든 피처를 수집한 뒤, 중복 여부를 분석했습니다. 80%가 겹친다는 사실을 확인하고, 중복을 제거한 뒤 고유한 4개 피처만 추가하여 최종 59개 피처셋을 구성했습니다. "무조건 합치는 것"이 아니라 **검증된 피처만 선별하는 전략**이 과적합을 방지하는 핵심이었습니다.

### 30분 발표를 기획하고 진행했습니다

노트북의 모든 과정을 33장 슬라이드 구조로 재구성하고, 각 슬라이드별 발표 스크립트를 작성했습니다. 단순한 결과 나열이 아니라 **"왜 이걸 했는가"라는 의사결정 과정**을 중심으로 스토리텔링 방식으로 구성했습니다.

## 프로젝트를 진행하며 마주한 난제들

### 난제 1: 타겟의 71%가 0인 극단적 분포

제주도 버스 정류장 10곳 중 7곳에서는 저녁에 아무도 타지 않습니다. 평균은 1.24명인데 최대값은 272명. 이 상태로 학습하면 모델이 "0이라고 하면 71% 맞겠네"라고 학습해버리는 문제가 있었습니다.

**해결**: log1p 변환으로 0~272 범위를 0~5.6으로 압축하여 모델이 0명인 곳도, 100명인 곳도 골고루 학습하도록 했습니다. 예측 후에는 expm1으로 원래 크기로 복원합니다.

### 난제 2: 9월로 학습해서 10월을 예측해야 하는 시간적 갭

학습 데이터는 9월인데, 예측 대상은 10월입니다. 추석 연휴가 끝나고 개천절/한글날이 오고, 날씨도 달라집니다. 단순히 9월 패턴을 외우면 10월에서 실패합니다.

**해결**: 시간 기반 교차검증(9/1~22 학습, 9/23~30 검증)으로 "미래 예측" 관점의 하이퍼파라미터 튜닝을 적용했습니다. 이 방식이 랜덤 KFold보다 10월 일반화에 효과적이었습니다.

### 난제 3: RMSE 대회인데 MAE 손실함수를 써야 했던 판단

대회 평가지표는 RMSE이지만, RMSE 손실로 학습하면 272명 같은 큰 값에 모델이 과도하게 끌려갔습니다. 데이터의 71%가 0인 상황에서는 이상치에 덜 민감한 MAE가 전체적으로 안정적인 예측을 만들었고, 결과적으로 RMSE 점수도 더 낮아졌습니다.

**교훈**: 평가지표와 손실함수가 반드시 같을 필요는 없다. 데이터 특성에 맞는 손실함수를 선택하는 것이 중요하다.

### 난제 4: test에만 있는 정류장 38개 (Cold Start)

10월에 새로 생긴 정류장 38개는 9월에 한 번도 등장하지 않아서, Target Encoding 값을 계산할 수 없었습니다.

**해결**: K-Fold Smoothed TE 함수가 자동으로 전역 평균(1.24명)으로 fallback하도록 설계되어 있어서, 별도 처리 없이 안전하게 동작했습니다. BTS 피처는 "기록 없음 = 이용자 0명"이므로 0으로 채웠습니다.

## 24번의 실험에서 얻은 핵심 교훈

### "피처를 늘리면 성능이 오를 것이다" — 틀렸습니다

| 실험 | 피처 수 | 모델 | Private 결과 |
|------|---------|------|-------------|
| plan9 | **55개** | LGB only | **단독 최고** |
| plan10 | 57개 | LGB only | 하락 |
| plan11 | 60개 | LGB only | 하락 |
| plan13 | 80개 | LGB only | 하락 |
| plan16 | 95개 | LGB only | 하락 |

55개에서 95개로 피처를 거의 2배 늘렸지만, **7번 연속 성능이 하락**했습니다. 41만행 규모의 데이터에서 피처를 무작정 늘리면 과적합이 발생한다는 것을 체감했습니다.

### "모델을 다양화하면 성능이 오른다" — 맞았습니다

| 실험 | 피처 수 | 모델 | Private 결과 |
|------|---------|------|-------------|
| plan9 | 55개 | LGB only | 기준점 |
| plan17 | 95개 | LGB + **RF** | 개선 |
| plan18 | 55개 | LGB + RF | **더 개선** |
| plan22 | 55개 | LGB + RF + **XGB** | **최고점** |

같은 피처인데 모델만 추가했더니 성능이 올랐습니다. LightGBM, RandomForest, XGBoost는 학습 방식이 다르기 때문에 서로의 약점을 보완합니다. 이 발견이 최종 3-way 블렌딩 전략의 기반이 되었습니다.

## 팀 협업: 피처 통합 과정

### 5명이 독립적으로 같은 결론에 도달했습니다

팀원 5명이 각자 만든 피처를 모아서 비교했더니, **핵심 아이디어의 80%가 겹쳤습니다.**

| 공통 개념 | 만든 팀원 수 | 우리 코드 변수명 |
|-----------|------------|----------------|
| 정류장 Target Encoding | 3명 | `te_station` |
| 아침 승차 합계 | 4명 | `ride_total` |
| BTS 아침 하차 | 1명 (핵심 기여) | `bts_morning_getoff` |
| 배차간격 | 1명 (핵심 기여) | `bus_interval` |
| 순유입 (승차-하차) | 1명 (핵심 기여) | `net_flow_6_8` |
| 승하차 비율 | 2명 | `ride_takeoff_ratio` |

5명이 서로 상의하지 않고 각자 분석했는데 같은 결론에 도달했다는 것은, 그 방향이 올바르다는 강력한 검증입니다.

### 중복을 제거하고 고유 피처만 추가했습니다

| 고유 피처 | 담당 | 설명 | 효과 |
|----------|------|------|------|
| `usertype_tourist` | 팀원 4 | 관광객 비율 | 관광 vs 통근 정류장 구분 |
| `usertype_local` | 팀원 4 | 현지인 비율 | 안정적 통근 수요 식별 |
| `dist_to_dong` | 팀원 4 | 동 중심까지 거리 | 도심 접근성 |
| `passed_days` | 팀원 5 | 경과일수 | 시간 흐름 반영 |

기존 55개 + 고유 4개 = **최종 59개**. "무조건 합치기"가 아니라 "검증된 것만 선별"하는 전략이었습니다.

## EDA에서 피처까지의 연결

근거 없이 만든 피처는 노이즈가 됩니다. 모든 피처는 EDA에서 발견한 인사이트에 기반합니다.

| EDA 발견 | 만들어진 피처 | 논리 |
|----------|-------------|------|
| 평일/주말 승차 패턴이 완전히 다름 | `is_weekend`, `dayofweek` | 주말엔 출퇴근 수요 사라짐 |
| 오전 승차와 저녁 승차의 상관 0.622 | `ride_total`, `ride_6_8` | 아침에 바쁜 곳 = 저녁에도 바쁨 |
| 아침에 내린 사람 = 저녁에 타는 사람 | `bts_morning_getoff` | 출근 때 내림 → 퇴근 때 탐 |
| 승차 - 하차 = 방향성 | `net_flow_6_8` | 음수 = 직장/학교 → 저녁 승차 높음 |
| 관광객 vs 직장인 패턴 다름 | `usertype_tourist` | 이용자 유형별 수요 차이 |
| 태풍(9/22~23) 때 승차 급감 | `is_typhoon` | 극단적 날씨 이벤트 처리 |
| 제주시 중심부에 수요 집중 | `dist_to_dong` | 도심 거리 = 수요 크기 |

## 최종 전략: 3-Way 블렌딩

```
최종 제출 (Private 13위) =
    55개 피처 모델 × 0.5    안정적 베이스 (과적합 없음)
  + 59개 피처 모델 × 0.2    이용자 유형 정보 보완
  + 95개 피처 모델 × 0.3    다른 결정 경계로 다양성 확보
```

### 각 모델의 역할

| 모델 | 피처 | 구성 | 역할 |
|------|------|------|------|
| 55개 모델 | 핵심 피처만 | LGB + RF + XGB × 3시드 | 안정적 예측 (비중 50%) |
| 59개 모델 | 55 + 팀원 고유 4개 | LGB + RF + XGB × 3시드 | 관광/통근 구분 보완 (비중 20%) |
| 95개 모델 | 대량 피처 | LGB + RF | 세밀한 패턴 포착 (비중 30%) |

**최고 점수는 하나의 강력한 모델이 아니라, 서로 다른 관점을 가진 여러 모델의 조합에서 나왔습니다.**

## 발표 준비 과정

### 33장 슬라이드를 "왜 이걸 했는가" 중심으로 구성했습니다

단순히 "이렇게 했습니다"가 아니라, 매 단계마다 **의사결정 과정**을 설명하는 방식으로 발표를 구성했습니다.

- EDA의 모든 차트에 "이 발견에서 어떤 피처가 나왔는가"를 연결
- 모델링 실패 과정(7번 연속 하락)을 숨기지 않고, 그 과정에서 얻은 교훈을 강조
- 예상 질문 15개에 대한 답변을 미리 준비 (보충 슬라이드 S29~S31)

### 발표 구조

| 구간 | 슬라이드 | 시간 | 핵심 메시지 |
|------|----------|------|-----------|
| 도입 | S1~S4 | ~6분 | 프로젝트 개요 + 타겟 분포의 극단성 |
| EDA | S5~S15 | ~12분 | 5가지 질문으로 정리한 데이터 이해 |

| 마무리 | S27~S33 | ~4분 30초 | 결과 + 인사이트 5가지 |

## 이 프로젝트에서 성장한 점

1. **데이터 분포를 이해하는 것이 모든 의사결정의 출발점**임을 체감했습니다. 71%가 0인 분포를 이해하지 못했다면 log1p 변환도, MAE 손실함수 선택도 할 수 없었습니다.

2. **"더 많이 = 더 좋다"는 직관이 틀릴 수 있다**는 것을 24번의 실험으로 검증했습니다. 피처 수를 늘리는 것보다 모델의 다양성을 확보하는 것이 더 효과적이었습니다.

3. **팀 협업에서 "독립적으로 같은 결론에 도달하는 것"의 가치**를 배웠습니다. 5명이 80% 겹치는 피처를 만든 것은 방향이 올바르다는 가장 강력한 검증이었습니다.

4. **실패를 기록하고 분석하는 것의 중요성**을 깨달았습니다. plan10~16의 7번 연속 실패가 없었다면, "피처 < 모델 다양성"이라는 핵심 교훈을 얻지 못했을 것입니다.

5. **발표는 결과가 아니라 과정을 보여주는 것**이라는 점을 경험했습니다. "13위를 달성했습니다"보다 "왜 이 전략을 선택했고, 어떤 시행착오를 거쳤는가"가 더 설득력 있었습니다.
